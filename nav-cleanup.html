Why This Matters

The Main Snapshot snapshot includes:

  • 12 service pages
  • 14 service area pages

Most clients don't use all of them. Unused pages with placeholder or irrelevant content look unprofessional and hurt SEO.

Rule: Only show pages relevant to the client's actual services and areas.

Process

Step 1: Identify What Client Offers

From onboarding form, note:

  • Actual services they provide
  • Actual areas they serve

Step 2: Compare to Generated Content

Check Custom Values:

  • service_1 through service_12
  • biz_area_1 through biz_area_14

Mark which ones are:

  • Real services/areas - Keep
  • Filler/unused - Remove from nav

Step 3: Edit Navigation Menu

  1. Go to Sites - [Website] - Edit
  2. Click on the navigation menu (header)
  3. Find the "Services" dropdown
  4. Remove links to unused service pages
  5. Find the "Areas" dropdown
  6. Remove links to unused area pages
  7. Save changes

Step 4: Consider Hiding Pages

Option A: Just remove from nav (recommended)

  • Page still exists
  • Search engines may still index
  • Easier to add back later

Option B: Unpublish page

  • Go to Sites - Pages
  • Click the page
  • Toggle "Published" off
  • Page returns 404

Use Option A unless the page has bad/placeholder content.

Navigation Structure Guidelines

Services Menu

If client has 1-3 services:

  • List them directly in main nav (no dropdown)

If client has 4-12 services:

  • Use "Services" dropdown
  • List all in dropdown

Example:

Home | Services v | Areas v | About | Contact
         +-- Drain Cleaning
         +-- Water Heater Repair
         +-- Pipe Repair

Areas Menu

If client has 1-3 areas:

  • List them directly OR skip areas dropdown entirely

If client has 4-14 areas:

  • Use "Service Areas" dropdown
  • List all in dropdown

Example:

Home | Services v | Service Areas v | About | Contact
                        +-- San Diego
                        +-- La Jolla
                        +-- Main Snapshot

Common Scenarios

Client only offers 4 services (of 12)

  1. Check which 4 services are real
  2. Remove other 8 from nav dropdown
  3. Optionally unpublish the 8 unused pages

Client only serves 5 areas (of 14)

  1. Check which 5 areas are real
  2. Remove other 9 from nav dropdown
  3. Optionally unpublish unused area pages

Client doesn't want area pages at all

  1. Remove entire "Service Areas" dropdown from nav
  2. Keep or unpublish area pages (your choice)

Client wants to add a service later

  1. Add the page back to navigation
  2. Regenerate content for that service using Selective Regen
  3. QA the new page

Editing Navigation in GHL

Method 1: Visual Editor

  1. Sites - [Website] - Edit
  2. Click the header/nav section
  3. Click on the menu element
  4. Edit menu items directly
  5. Delete unwanted items
  6. Save

Method 2: Site Settings

  1. Sites - [Website] - Settings
  2. Find Navigation/Menu settings
  3. Edit there (if available)
Note: GHL's editor can be finicky. Sometimes you need to click precisely on the menu element.

QA After Cleanup

After cleaning navigation:

  • Click every nav link - all work
  • No broken links in nav
  • No placeholder pages visible in nav
  • Mobile nav works correctly
  • Dropdown menus function properly

Checklist

  • Reviewed client's actual services
  • Reviewed client's actual service areas
  • Removed unused services from nav
  • Removed unused areas from nav
  • Tested all nav links
  • Tested mobile nav
  • Saved changes
