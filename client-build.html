Step-by-Step Tasks

Day 0: Deal Won

AUTOMATIC:

  • SaaS Configurator creates sub-account from Carlsbad snapshot
  • WF-OB1: Welcome SMS + onboarding form link
  • WF-OB2: Reminder loop starts (48h intervals)

HUMAN: Nothing - wait for form

Day 1-3: Form Submitted

AUTOMATIC:

  • Form data saved, WF-OB3 sends GMB request SMS
  • Content Generator saves to Baserow, scrapes website
  • Status: awaiting_notes

HUMAN TASKS (all parallel):

Task 1: Review Form Completeness (2 min)

Check onboarding form for:

  • Required fields filled (name, business, services, areas, hours)
  • Services make sense for niche
  • Logo uploaded OR "needs logo" checked
  • Contact info complete

If incomplete: SMS client for missing info

Task 2: Purchase Domain (5 min)

  1. Choose domain: [businessname].com (no LLC/Inc)
  2. Purchase from Namecheap or Cloudflare (~$10-15/year)
  3. In client sub-account: Settings - Domains - Add Domain
  4. Add DNS records in registrar
  5. Wait for SSL (5-30 min)

Task 3: Buy Phone Number (2 min)

Check if client has EIN:

Client HasBuy ThisRegistration Type
EIN providedLOCAL number (area code)10DLC brand + campaign
No EIN (sole proprietor)TOLL-FREE numberToll-free verification
  1. Settings - Phone Numbers - Add Number
  2. Filter appropriately (local or toll-free)
  3. Purchase

Task 4: Submit Build Notes (2 min)

Triggers Content Generator to run:

Build Notes Form:
https://api.slatesystems.io/widget/form/O07QiRk5BZkddn7DgKI1
  1. Open the Build Notes form (link above)
  2. Enter location_id
  3. Add any corrections/notes
  4. Submit - AI generates ~282 custom values

After Tasks 1-4: Update Pipeline

  • Pipeline: Client Delivery → A2P In Progress
  • Tag: a2p-pending

Day 2-8: Building

AUTOMATIC:

  • WF-OB4: Progress SMS sequence
  • Content Generator runs (after Build Notes)

HUMAN TASKS:

Task 5: Submit A2P (10 min) - NEEDS PHONE FIRST

See A2P Submission SOP for full process.

For LOCAL numbers: Full 10DLC registration (EIN must match exactly)

For TOLL-FREE: Simpler verification

Task 6: Configure Phone Forwarding (2 min) - AFTER A2P SUBMITTED

  1. Phone Numbers - [number] - Edit Configuration
  2. Forward Calls To: client's business phone (+1)
  3. Disable call connect
  4. Enable "Ring All Users"
  5. Save

Task 7: Verify AI Content (5 min) - PARALLEL

  1. Go to client sub-account - Settings - Custom Values
  2. Spot-check: company_name, services, areas, colors
  3. Preview website
  4. If issues: use Selective Regen form

Task 8: Process Logo (5-30 min) - PARALLEL

  • Uploaded: Download, resize 400x400, upload to media
  • Needs creation: Use Canva, match industry + colors

See Logo Processing SOP for full process.

Task 9: Configure Chat Widget (5 min) - PARALLEL

  1. Sites - Chat Widget
  2. Colors: use color_secondary for all
  3. Welcome: "Shoot me any questions..."
  4. Avatar: client logo
  5. CTA: "Send :)"
  6. Disable agency branding
  7. Enable on website: Sites - [Website] - Settings - Chat Widget

Task 10: Update Navigation (5 min) - PARALLEL

  1. Sites - [Website] - Edit
  2. Remove services client doesn't offer
  3. Remove areas client doesn't serve

See Navigation Cleanup SOP for full process.

Day 5-9: GMB Connection

When client grants access to hello@slatesystems.io:

Task 11: Connect GMB to GHL (5 min) - FIRST

  1. Settings - Integrations - Sign in with Google
  2. Use hello@slatesystems.io
  3. Select client's business
  4. Connect

Task 12: Update GMB (5 min) - AFTER CONNECT

  1. Business Profile Manager - Edit Profile
  2. Phone: [purchased number]
  3. Website: https://www.[domain].com (with www!)
  4. Save

Task 13: Get GMB Values (5 min) - AFTER UPDATE

Custom ValueHow to Get
company_gmb_linkCopy GMB profile URL
company_google_review_linkGMB - Ask for reviews
google_map_embed_codeGoogle Maps - Share - Embed (450x400)

See GMB Setup SOP for complete process.

Task 14: Verify Reviews AI (3 min) - PARALLEL

  • Reputation - Settings - Reviews AI
  • Confirm "Positive Review Responder" exists (from snapshot)

Task 15: Configure Social Planner (3 min) - PARALLEL (REQUIRED)

  1. Marketing - Social Planner
  2. Content - New Post - Post Reviews
  3. Source: Google, Stars: 5 only, Account: client GMB
  4. Frequency: Weekly
  5. Skip only if: zero existing reviews

Task 16: Create Review Widget (3 min) - PARALLEL

  1. Reputation - Widgets - Create New
  2. Layout: Carousel, 5 Star Only
  3. Colors: all transparent/white
  4. Save - Make LIVE
  5. Embed in homepage Reviews section

Task 17: Add Schema Markup (2 min) - PARALLEL

Sites → [Website] → Settings → Head Tracking Code:

Choose @type based on niche:

  • Painters, Roofers, General Contractors: HomeAndConstructionBusiness
  • Plumbers, HVAC: Plumber or HVACBusiness
  • Electricians: Electrician
  • Landscapers: LandscapingBusiness
  • Other: LocalBusiness
<script type="application/ld+json">{
  "@context": "https://schema.org",
  "@type": "[CHOOSE_TYPE]",
  "name": "{{custom_values.company_name}}",
  "url": "{{custom_values.company_website_link}}",
  "telephone": "{{custom_values.company_twilio_phone}}"
}</script>

Day 8-9: QA Review

Task 18: Run QA Checklist (15 min)

See Pre-Launch Checklist

Key checks:

  • Website on custom domain + SSL
  • Phone forwarding works
  • Chat widget works
  • Review funnel works
  • Review Widget displays
  • Schema in head
  • Social Planner configured
  • No placeholder text
  • Navigation clean
  • Mobile responsive

Task 19: Fix Issues

For each QA issue found:

  1. Content issues (wrong text, placeholder, typos): Fix directly in Custom Values or use Selective Regen
  2. Phone issues: Check forwarding config, verify number active
  3. Chat widget issues: Re-check Sites → Chat Widget settings
  4. Review widget issues: Verify GMB connected, widget is LIVE
  5. SSL/Domain issues: Check DNS records, wait for propagation (up to 48h)
  6. Navigation issues: Use Nav Cleanup SOP

Re-run QA checklist after fixes to verify resolution.

Task 20: Move to Launch

  • Pipeline: - Launch Scheduled
  • WF-OB4 sends launch call invite automatically

Day 9-14: Launch Call

Closer handles - see Launch Call SOP in Sales Portal

After launch:

  • Pipeline: - Active
  • WF-OB6 handles post-launch sequence
  • Tag: currentclient added automatically

Troubleshooting

A2P Rejected

  1. Check rejection reason (usually EIN name/address mismatch)
  2. Resubmit with exact EIN info
  3. If stuck: GHL ticket - Book with Gabby
  4. Push back billing if needed

Client Won't Grant GMB

  1. Resend guides.slatesystems.io link
  2. Offer screenshare call
  3. If no GMB: guide through creation + verification
  4. If claimed by someone else: recovery process (7+ days)

Content Needs Correction

Use Selective Regeneration:

  1. Go to regen.slatesystems.io
  2. Enter location_id
  3. Select fields to fix
  4. Add feedback notes
  5. Regenerate
