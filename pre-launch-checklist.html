Pre-Launch Checklist (Complete)

Role: Fulfillment Specialist
When: Before moving opportunity to "Launch Scheduled"
Time: 20-30 minutes per client
Last Updated: Feb 22, 2026

Overview

This checklist combines all QA checks needed before scheduling a client's launch call. Complete all sections before moving the opportunity.

1. SUB-ACCOUNT SETUP

  • Sub-account created from Carlsbad snapshot
  • Business name correct in sub-account settings
  • Reselling disabled in Reselling tab
  • Webhook URL registered in Baserow (for future regenerations)
  • All 16 Carlsbad workflows are active
  • No error indicators in workflow editor

2. PHONE & SMS

Toll-Free Number

  • Number purchased
  • Phone forwarding configured to client's business phone
  • Test call: rings client's phone

A2P Verification

  • A2P verification submitted
  • A2P status: _____________ (pending/approved/rejected)
Note: If A2P pending > 7 days, note in pipeline and escalate. If approved, ready to send automated texts. If pending, client can still receive calls but auto-texts won't send until approved.

Missed Call Text-Back

  • Workflow active
  • Test: call toll-free, don't answer, verify text received

3. CONTENT GENERATION

Custom Values - AI Generated (Verify populated)

  • company_name — No LLC/Inc
  • broad_service_name — General niche term
  • service_1 through service_12 — Match client's actual services
  • biz_area_1 through biz_area_14 — Match client's coverage
  • color_primary — Looks good on site
  • color_secondary — Complements primary
  • about_us text — Professional, accurate

Custom Values - Human Entered (Must fill manually)

  • company_phone__aesthetic_ — (XXX) XXX-XXXX format
  • company_phone__functional_ — +1XXXXXXXXXX format
  • company_twilio_phone — Same as functional
  • company_website_link — www.[domain].com
  • website_terms_page_link — /terms

Content Quality

  • No placeholder text visible ("Lorem ipsum", "[SERVICE]", "{{")
  • Content regenerated if issues found (use Selective Regen)

4. GMB (GOOGLE BUSINESS PROFILE)

If Client Has GMB

  • Client granted GMB access
  • GMB connected in GHL Settings → Integrations
  • GMB phone updated to toll-free
  • GMB website updated to new domain (with www.)
  • company_gmb_link — Full profile URL
  • gmb_review_link — g.page/r/... format
  • google_map_embed — iframe code (width=450, height=400)

If Client Has No GMB

  • Leave GMB values blank
  • Note for launch call: "No GMB - client needs to set up"

5. WEBSITE QA

Domain & SSL

  • Domain connected (not GHL subdomain)
  • SSL certificate active (padlock shows)
  • Site loads on https://www.[domain].com

Homepage

  • Logo displays correctly
  • Company name correct
  • Phone number correct (toll-free)
  • Colors match brand / look professional
  • Hero background: black + Half Fade (white text readable)
  • Quote form works (test submission)
  • Chat widget appears and works
  • Review Widget displays (carousel with 5-star reviews)
  • Reviews section: black + Half Fade (if has bg image)
  • Quote CTA section: black + Half Fade (if has bg image)

Navigation

  • All menu items work
  • No broken links
  • Services listed match client's actual services
  • Unused service pages removed from nav
  • Unused area pages removed from nav

Service Pages

  • Each service page has content
  • Service images load (3 per service)
  • Hero background: black + Half Fade
  • No placeholder text

Service Area Pages

  • Areas match client's coverage
  • Hero background: black + Half Fade
  • No placeholder text

Blog Posts

  • At least 6 blog posts exist
  • Content is relevant to business niche
  • No placeholder text

Footer

  • Contact info correct
  • Social links work (if provided)
  • Privacy policy link works
  • Terms of service link works

Mobile

  • Site loads on mobile
  • Navigation works
  • Forms usable
  • Chat widget accessible
  • No horizontal scrolling issues

Images

  • Hero image loads (HTTPS, not HTTP)
  • Service page images load
  • Area page images load
  • Gallery images load
  • About section image loads
  • No broken image placeholders

Schema Markup

  • View page source → search for "application/ld+json"
  • Schema includes company name, phone, address

6. AUTOMATIONS & INTEGRATIONS

Review System

  • Review funnel accessible
  • Test: 5-star redirects to Google
  • Test: 3-star goes to private feedback
  • Marketing form in app works (Marketing → Forms)

Chat Widget

  • Widget appears on all pages
  • Colors match brand (color_secondary)
  • Welcome message set
  • Client logo as avatar
  • Test message → appears in GHL inbox
  • Auto-reply sends

Quote Form

  • Form submits successfully
  • Lead appears in GHL
  • Notification workflow triggers

Reviews AI

  • Reviews AI agent exists (Reputation → Settings → Reviews AI)
  • Agent configured for 4-5 star reviews
  • GMB connected as source
  • Wait time: 5 minutes

Social Planner (if client has existing reviews)

  • Social Planner configured (Marketing → Social Planner)
  • Source: Google reviews
  • Filter: 5-star only
  • Frequency: Weekly

Review Widget

  • Widget created (Reputation → Widgets)
  • Layout: Carousel
  • Filter: 5-star only
  • Widget embedded in homepage Reviews section

7. USER SETUP

  • Client user created in sub-account
  • Role: Admin
  • Email: [client's business email]
  • Phone number added
  • Password set (with special character for app login)

8. FINAL CHECKS

  • No "Stone Systems" or old branding anywhere
  • No test data in inbox/contacts (delete test submissions)
  • Client contact info in opportunity is correct
  • Launch call availability noted (check client timezone)

  3. Notify Closer that client is ready
